# Contract: Message Confidence (REST, additive to conversations module)

Additive, non-breaking extension of the timeline `Message` object (`GET /tenant/conversations/{id}/messages`, `AddMessageResponse.message`, and the `ai.message.completed` SSE payload). Existing consumers that ignore the new field are unaffected (Principle V).

## Additive field: `confidence`

```jsonc
{
  "id": "…",
  "kind": "ai",
  "body": "Our enterprise plan includes SSO and a dedicated CSM.",
  "created_at": "2026-07-18T10:00:00Z",
  "citations": [ /* 020 contract, unchanged */ ],
  "confidence": {
    "score": 0.81,              // stored ai_confidence_score, 0–1
    "band": "high"              // derived server-side: high ≥ 0.70, medium ≥ 0.40, low < 0.40
  }
}
```

## Contract rules

1. **AI messages only**: `confidence` is present (non-null) only on `kind: "ai"` messages generated by the 021 engine. It is `null`/absent for all other kinds and for AI messages created before this feature.
2. **Band derivation is server-owned**: clients render `band` as-is (badge) and MUST NOT re-derive it from `score`; thresholds may be tuned server-side without a client release. `score` is provided for display detail (e.g., tooltip) and future tooling.
3. **Staff-only surface**: this field appears only on authenticated tenant dashboard routes. Any future customer-facing message contract (channel widgets) MUST omit it (FR-010).
4. **Fallback messages** (`kind: "system"`) never carry `confidence` or `citations`.

## Internal interfaces touched (not REST)

- `conversations::model::Message` gains `confidence: Option<ConfidenceView>` (`score: f32`, `band: &'static str`-serialized enum).
- `conversations::queries::insert_ai_reply_in_tx` gains the score parameter; timeline/detail queries select the new column (no N+1 — same row).
- Band function: single `pub fn confidence_band(score: f32) -> Band` in `modules/ai::confidence`, re-exported for the conversations view mapping.
